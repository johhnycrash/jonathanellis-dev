Backend architecture

What needs to be built next session

Small Flask app deployed alongside Coolify. ~150 lines of Python. Endpoints below. The frontend you're seeing already calls these — once the backend is live, this page works.

GET  /api/domains
     → calls Hestia: v-list-mail-domains server-ops json
     → returns: ["jonathanellis.dev", "jonathanellis.ca", "teamabsolute.ca"]

POST /api/mailbox
     body: { domain, local, password }
     → calls Hestia: v-add-mail-account server-ops <domain> <local> <pw> 0
     → returns: { ok: true, email: "<local>@<domain>" }

GET  /api/profile/<email>
     → generates a .mobileconfig from a template
     → returns: application/x-apple-aspen-config download

POST /api/domain
     body: { domain }
     → 1. v-add-mail-domain server-ops <domain>
     → 2. read /home/server-ops/conf/mail/<domain>/dkim.pub.key
     → 3. Cloudflare API: add MX, SPF, DKIM, DMARC records
     → 4. certbot certonly --dns-cloudflare ... -d mail.<domain> -d webmail.<domain> (re-issue multi-SAN cert)
     → 5. reload exim4 + dovecot
     → returns: { ok: true, dns_added: [...], cert_issued: true }

All of this sits behind Cloudflare Access so only your authenticated email can reach jonathanellis.dev/newemailaccount in the first place. The Flask backend trusts the CF-Access-Authenticated-User-Email header set by Cloudflare — no second auth layer needed.
