Health check

Run this first when something seems off. One copy-paste shows you whether mail, web, and SSL are all happy.

ssh root@178.105.40.71 'echo "=== Services ==="; systemctl is-active dovecot exim4 hestia docker; echo; echo "=== Listening ports ==="; ss -tlnp | grep -E ":(25|80|443|587|993|8083|8000)\s" | awk "{print \$4, \$NF}"; echo; echo "=== Disk ==="; df -h / | tail -1; echo; echo "=== Memory ==="; free -h | grep Mem; echo; echo "=== Mail queue ==="; exim -bpc'

Expected output: dovecot/exim4/hestia/docker all active, ports 25/80/443/587/993/8083/8000 listening, mail queue near 0.

Add a new email domain

Full procedure for hosting a new domain (e.g., a client's). Repeat for every domain you onboard.

1. Add the mail domain in HestiaCP

Log into HestiaCP as server-ops → Mail tab → Add Mail Domain. Set:

  • Domain: example.com
  • Webmail Client: Roundcube (default)
  • DKIM Support: ✓
  • Spam Filter ✓, Reject Spam ✓, Anti-Virus ✓
  • SMTP Relay: unchecked (inherits global Brevo)
  • Enable SSL: unchecked (will fail — apache2 is stopped; not needed since cert is server-wide)

Save. Mailbox account creation comes after DNS.

2. Get the DKIM public key

ssh root@178.105.40.71 echo "v=DKIM1; k=rsa; p=$(openssl rsa -in /etc/exim4/domains/example.com/dkim.pem -pubout -outform der 2>/dev/null | openssl base64 -A)"

Replace example.com with the actual domain. Output is the full TXT record value for mail._domainkey. Copy it.

3. Authenticate the domain in Brevo

Brevo → Settings → Senders, Domains & IPs → Domains → Add a Domain. Enter example.com. Brevo gives you 2 CNAMEs and 1 verification TXT record to add to DNS.

4. Add DNS records at the domain's DNS host

If DNS is at Cloudflare (most cases). Add ALL of these as DNS only (grey cloud):

TypeNameContentPriority
MX@mail.jonathanellis.dev.10
TXT@v=spf1 +mx include:spf.brevo.com ~all—
TXT_dmarcv=DMARC1; p=quarantine; adkim=r; aspf=r; rua=mailto:hello@jonathanellis.ca;—
TXTmail._domainkey(value from Step 2)—
CNAMEbrevo1._domainkey(value from Brevo, like b1.example-com.dkim.brevo.com)—
CNAMEbrevo2._domainkey(value from Brevo)—
TXT@brevo-code:... (value from Brevo)—

Wait 1–2 minutes for propagation. Click Verify in Brevo — all records should turn green.

5. (Optional) Expand SSL cert to cover mail/webmail.example.com

Only if you want clients connecting via mail.example.com instead of mail.jonathanellis.dev. Skip if you're routing all mail clients to mail.jonathanellis.dev.

Add CNAMEs first:

TypeNameContent
CNAMEmailmail.jonathanellis.dev
CNAMEwebmailmail.jonathanellis.dev

Then expand the cert (see SSL — renew/expand section, just add the new domains to the -d list).

6. Test

Send a test email from a mailbox on the new domain (created next) to a Gmail address. In Gmail, "Show Original" — check spf=pass, dkim=pass, dmarc=pass.

Migrate mail in

Use imapsync to copy mail from any IMAP server (GoDaddy, MXroute, anywhere) into a mailbox on this server. Idempotent — running it twice doesn't duplicate messages.

Install imapsync on the server (one-time)

ssh root@178.105.40.71 apt update && apt install -y imapsync

Full sync — copy everything from a source server

Replace the SOURCE_* variables with the source IMAP details and DEST_PASS with the new mailbox password.

imapsync \ --host1 SOURCE_HOST_OR_IP \ --port1 993 --ssl1 --nosslcheck1 \ --user1 USER@example.com \ --password1 'SOURCE_PASSWORD' \ --host2 mail.jonathanellis.dev \ --port2 993 --ssl2 \ --user2 USER@example.com \ --password2 'DEST_PASSWORD' \ --automap \ --no-modulesversion

Incremental sync — only new messages since a date

Add --maxage N (sync only messages from the last N days) or --minage N (skip messages older than N days). Useful when you've already done a bulk sync and want to grab anything new since.

# Only messages from the last 14 days imapsync \ --host1 SOURCE_HOST_OR_IP --port1 993 --ssl1 --nosslcheck1 \ --user1 USER@example.com --password1 'SOURCE_PASSWORD' \ --host2 mail.jonathanellis.dev --port2 993 --ssl2 \ --user2 USER@example.com --password2 'DEST_PASSWORD' \ --automap --no-modulesversion \ --maxage 14

Dry-run first (always)

Add --dry to any command to preview what would happen without copying anything. Run dry, scroll output, confirm folder counts look right, then run for real.

Why Brevo

Hetzner blocks port 25 outbound on new accounts (anti-spam). Brevo relays our outbound mail through their reputable IPs. Bonus: better deliverability than direct from a fresh VPS.

SSL — renew or expand

Cert covers (currently)

  • server.jonathanellis.dev
  • mail.jonathanellis.dev
  • webmail.jonathanellis.dev
  • mail.jonathanellis.ca
  • webmail.jonathanellis.ca

Stored at /etc/letsencrypt/live/hestia-mail-panel/ as fullchain.pem + privkey.pem.

Add new hostnames to the cert

Use this when adding mail hosting for a new domain. Just append -d new.example.com for each new hostname:

ssh root@178.105.40.71 certbot certonly \ --dns-cloudflare \ --dns-cloudflare-credentials /root/.secrets/cloudflare.ini \ --dns-cloudflare-propagation-seconds 30 \ -d server.jonathanellis.dev \ -d mail.jonathanellis.dev \ -d webmail.jonathanellis.dev \ -d mail.jonathanellis.ca \ -d webmail.jonathanellis.ca \ -d mail.NEWDOMAIN.com \ -d webmail.NEWDOMAIN.com \ --email hello@jonathanellis.ca \ --agree-tos --non-interactive \ --cert-name hestia-mail-panel \ --expand systemctl restart dovecot exim4 hestia
If certbot errors with "Unable to determine zone_id": the Cloudflare API token at /root/.secrets/cloudflare.ini doesn't have access to the new zone. In Cloudflare → My Profile → API Tokens → edit the token → Zone Resources → "All zones from an account". Save and retry.

Force renewal (bypass "not due yet")

certbot renew --cert-name hestia-mail-panel --force-renewal systemctl restart dovecot exim4 hestia

Auto-renewal hooks (already installed)

Located at /etc/letsencrypt/renewal-hooks/deploy/reload-mail.sh. Reloads exim4, dovecot, hestia after every renewal. Runs from systemd timer (twice daily).

Get DKIM public key for any domain

HestiaCP stores the private key as /etc/exim4/domains/DOMAIN/dkim.pem. To extract the public key in DNS-record format:

ssh root@178.105.40.71 echo "v=DKIM1; k=rsa; p=$(openssl rsa -in /etc/exim4/domains/DOMAIN/dkim.pem -pubout -outform der 2>/dev/null | openssl base64 -A)"

Replace DOMAIN with the actual domain. Output is one line, ready to paste into Cloudflare as TXT mail._domainkey.

Never share dkim.pem contents. That's the PRIVATE key. Only the derived public key (above) goes in DNS.

Common errors & fixes

"Error: ERROR: Restart of apache2 failed" in HestiaCP UI

Expected. Apache is intentionally stopped (Coolify needs the ports). The underlying mail config still saves correctly. Use CLI commands (v-change-mail-domain-catchall, etc.) when UI errors.

iPhone Mail: "Cannot verify server identity"

Cert isn't trusted yet. Causes:

  • You're using mail.jonathanellis.ca but cert wasn't expanded — use mail.jonathanellis.dev as server, OR run cert expand command
  • Self-signed cert was cached — delete account on iPhone, re-add fresh

iPhone Mail: "Server is not responding"

Network or service issue. Check:

systemctl status dovecot ss -tlnp | grep ':993'

If both look fine: try cellular instead of WiFi (some networks block 993).

Brevo: "sender is not valid" / mail accepted but never delivered

Sender domain isn't authenticated in Brevo. The sender domain (after the @) must be added to Brevo → Domains. Different from the destination domain.

Cloudflare DNS: "ERR_CERT_AUTHORITY_INVALID" on subdomain

Coolify Traefik is serving a self-signed default cert because no Coolify app is configured for that hostname. Either deploy something to that domain in Coolify, OR add it to the Hestia mail cert via the certbot expand command.

Coolify cert issuance fails ("DNS problem: NXDOMAIN")

Domain typo in Coolify. Check Projects → app → Configuration → Domains. Fix the typo, save, redeploy.

SSH: "Permission denied (publickey)"

Your SSH key changed or you're on a new machine. Add the new public key via Hetzner Cloud Console (rescue mode), or use a different machine that has the key.
