Notes Why it's set up this way

Why I changed Hestia's recommended SPF

Hestia suggested v=spf1 a mx ip4:178.105.40.71 -all — too strict. The -all hard-fails any sender that isn't your server's IP, which would block your Brevo relay (your outbound goes through Brevo's IPs, not the server). The version above authorizes both Brevo and Amazon SES (Resend) and uses soft-fail (~all) — same shape that works for jonathanellis.dev.

Why both DKIM selectors are needed (mail + resend)

You sign outbound mail twice: once by Hestia (selector mail) for IMAP/SMTP messages, and once by Resend (selector resend) for transactional. They live at different DNS names so there's no conflict — recipients verify whichever signature matches the sending path.

Brevo will likely want a third selector when you authenticate the domain there — typically mail2._domainkey. Add that record when Brevo gives it to you.

Cross-domain DMARC reporting — authorization record needed

Because teamabsolute.ca's rua reports go to rua@jonathanellis.dev (a different domain), per RFC 7489 you need an authorization record at the receiving domain. Add this TXT record to the jonathanellis.dev zone (not teamabsolute.ca):

Name: teamabsolute.ca._report._dmarc
Value: v=DMARC1

Without this, Google/Microsoft will refuse to send the DMARC reports across domains.

Mail records must be DNS-only (gray cloud)

Cloudflare proxy (orange cloud) only handles HTTP/S. Proxying an MX or A record meant for mail breaks SMTP delivery — receivers can't reach the actual server. Always gray for: mail, webmail, MX, all _domainkey, _dmarc, SPF.

If "nginx restart failed" comes back

Hestia's GUI tries to set up nginx for the webmail subdomain on ports 80/443, which Coolify owns. The mail domain itself adds fine — just the web-side fails. To silence:

v-delete-web-domain server-ops webmail.teamabsolute.ca 2>/dev/null

nginx -t && systemctl reload nginx

After When DNS has propagated

1. Issue Let's Encrypt cert for mail.teamabsolute.ca

Wait ~5 min after editing DNS, then SSH in:

ssh root@server.jonathanellis.dev

Then:

v-add-letsencrypt-domain server-ops teamabsolute.ca '' '' '' mail.teamabsolute.ca

2. Verify everything from outside the server

dig +short MX teamabsolute.ca → expect 10 mail.teamabsolute.ca

dig +short A mail.teamabsolute.ca → expect 178.105.40.71

dig +short TXT mail._domainkey.teamabsolute.ca → expect the DKIM key (one long line)

dig +short TXT _dmarc.teamabsolute.ca → expect the new p=quarantine record

3. Authenticate domain in Brevo

Brevo dashboard → Senders & Domains → Domains → Authenticate teamabsolute.ca. Brevo will give you 1–3 CNAME records (typically mail2._domainkey + a verification token). Add those to Cloudflare exactly as Brevo specifies. Without this, outbound mail via Brevo isn't DKIM-signed by Brevo — gmail/outlook will be more likely to spam-bin it.

4. Test deliverability

Send a test from matt@teamabsolute.ca to a fresh address from mail-tester.com — they email back a score out of 10. Target: 9.5+. If anything's missing they'll tell you exactly which DNS line.
